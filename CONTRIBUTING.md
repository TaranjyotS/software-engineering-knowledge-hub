# Contributing

Thanks for considering a contribution.

## Good contributions

- Add clear interview notes or examples.
- Improve explanations without removing useful context.
- Add coding questions with expected output or reasoning.
- Keep Markdown formatting readable and consistent.
- Avoid committing credentials, tokens, local `.env` files, or personal data.

## Suggested note format

```md
# Topic Name

## Quick Summary

## Key Concepts

## Interview Questions

## Examples

## Common Mistakes
```
